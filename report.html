Introduction

My Web site is a fictional local club for enthusiasts of LEGO, and specifically Star Wars LEGO enthusiasts.

My kids and I are LEGO enthusiasts and also Star Wars enthusiasts, and there are local clubs in our city for each of these interests.

We have attended local LEGO club activities as well as fantasy conferences featuing Star Wars.

I wanted to create a Web site for the intersection of LEGO and Star Wars, as I know there is a huge community of enthusiasts for both topics.

The structure of the Web site focuses on the main interests for LEGO and Star Wars enthusiasts, specifically:

  • The main landing page, with links to pages showcasing the following topics
  • Upcoming events, such as LEGO Star Wars build event, collectables events, cosplay events, etc.
  • MOCS / 'my own creation' highlights, showcasing LEGO Star Wars creativity
  • Reviews and highlights of current or upcoming LEGO Star Wars kits
  • Information on joining the club, where it meets, who organizes club activities, etc.

Accessibility

For accessibility, I implemented:

  1. All images include a descriptive 'alt' attribute, and also have been styled with height and width percentages to be responive when displayed on smaller screens
  2. For the Services page forms, I modified all the click buttons, radio buttons and input fields to be larger, so they are easier to see and navigate to.
  3. These forms all have validation checkers to ensure every form item is completed, and there is no time limit to completing the form
  4. For the text on all pages, the font size is responsive within the page, with the page elements also being responsive. This allows the text to be made larger as needed.
  5. Many hyperlinks ('a href') elements have been included on all pages, allowing a user to tab through these hyperlinks to navigate to all sections of all web site pages.
  6. Finally, I loaded the web site into an online colour contrast evaluation service (listed on the W3C site), and this confirmed the colours used on my web site passed.

Usability

  1. For the navigation bar and the footer on every page, the CSS grid functionality is used. In addition, grid functionality is used for many other sections on individual web pages.
  2. With the CSS grid functionality, I am able to define grid columns and rows adjust in size, and how the HTML elements within a grid item can be displayed as screen viewers increase or decrease in width.
  3. With the grib function, for the columns both the 'auto-fill' and 'auto-fit' keyword and the 'minmax' function were used, to ensure that when displaying on smaller screens the columns could shrink and wrap onto an additional row.
  4. For the MOCS web page that features many images, the '@media' rule was used to have certain HTML tags not display in smaller screens, since these tags would 'clutter' smaller screens
  5. The '@media' rule was also used to modify padding size based on screen size
  6. For most images, the web pages include both smaller scale and larger scale versions of each image, so that whichever scale is best for a screen can be displayed
  7. For all fonts, the font size is also automatically based on the viewport width (using 'vw')


Learning

The most significant function I focused on learning was the CSS grid functionality, to allow columns and rows of HTML elements to auto-adjust to different screen sizes. I had used this functionality for our group work, but not extensively.

From the group work, I knew how grids could work, but not in detail or how to modify all the different parameters that can be used for grids. Most significantly, I wanted to understand completely how 'auto-fill' versus 'auto-fit' work.

While searching for how CSS grid functionality works, I found a web site CSS-TRICKS that has a number of articles and tutorials about the subject, including an extensive guide to grids.

To learn how to use grids, I would read about a specific feature, see how it worked in the examples, and then I mostly needed to experiment with using that feature in my site.

With experimenting, it is very much an iterative process, involving adding code to the CSS file, and see how the web pages works directly in a browser. In many instances, it did not work as planned, and I needed to debug and modify my code.

What I am most proud about was having the idea myself of implementing nested grids. For my reviews page, I wanted to have each review of three columns be responsive, and also the whole section be responsive.

So I implemented the CSS grid fucntionality for both each 'review section' and also all four sections of reviews.


I also needed to learn how to implement forms, beyond just basic input fields. I had learned about JavaScript from my prior Coursera learning, and leverage this to extend my knowledge using online searches, with much knowledge coming from Stackoverflow.

With my online searching, I gradually built up my knowledge, learning about the different field types, the different input types, the different button types, and how all of these types can be modified.

As with the CSS grid functionality, I needed to experiment with adding and modifying each of the forms features that I added. Most of these functions were applied in the CSS file, to change the size and appearance of a form item.

Ultimately, I was able to ensure the form fields were larger, the buttons were larger and had interesting styles, the radio buttons were larger, and the text was well placed. All these features also ensured better web site accessibility.


Finally, for the forms on my Services page, I wanted to leverage JavaScript to make these forms interactive, with buttons providing messages after clicked, having the forms validated, and being able to create different types of forms.

I had learned about JavaScript from my prior Coursera learning, and leverage this to extend my knowledge using online searches, with much knowledge coming from Stackoverflow.

As with the CSS grid functionality, I needed to experiment with implementing JavaScript. What I focused on what adding JavaScript to the form buttons, to validate the input fields and to add a user's input to a list.

Ultimately, I was able to add a validation feature to the submit button that would only allow submission if all form fields were completed. The submission button would also then reset the form to be blank.

I was also able to implement a form that allows a user to enter a value, and this value would then be printed as an additional un-ordered list item, after the existing list items.

When looking in the developer tools, I would then see that the style had been 'inherited' from another CSS selector or HTML element. I was able to modity the code as needed, but I sometimes still do not understand why it was inherited.

I also want to work more with JavaScript. I have taken a number of Python courses, so I understand variables, types, conditionals, recursion and other aspects of coding that are re-useable in JavaScript.

My background coding knowledge was very useful with implementing similar logic to the JavaScript used in my Website.

But I really would like to learn more about how JavaScript works in terms of JavaScript semantics and coding logic.

Lastly, I want to have a better understanding of how a web page can be laid out in terms of HTML elements, such that when thinking about designing a web page or looking at a web page, instinctively knowning what elements needs to be used or are used.

I feel I have a good understanding already, but when I loaded each page into the W3C HTML Validation service, there were basic errors I had made, which were easy to fix and could have been avoided.
